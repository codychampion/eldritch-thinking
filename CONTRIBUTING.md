# Contributing

Keep additions small, vivid, and useful in real interfaces.

Guidelines:

- Prefer status lines that are funny without being unusable.
- Keep each line short enough for CLI output, dashboard chips, and loading states.
- Avoid direct references to copyrighted characters or modern franchise names.
- Preserve the leading-subject pattern where useful, especially lines beginning with `Claude`.
- Add new packs as plain JSON arrays under `statuses/`.
- Run `npm test` and `npm pack --dry-run` before opening a PR.
